# Contributing to ReceiptSorter

Thank you for your interest in contributing to ReceiptSorter! We welcome contributions from the community to help make this utility better for everyone.

## Contributor License Agreement (CLA)

To ensure that the project remains open-source and to protect both contributors and users, we require all contributors to sign our Contributor License Agreement (CLA).

**You can find the central CLA here: [OJD Technical Solutions CLA](https://github.com/Ic3burG/.github/blob/main/CLA.md)**

### Important Notes:
- **Sign Once:** Signing the CLA once covers all projects under OJD Technical Solutions.
- **Automated Check:** We use CLA Assistant to automatically check pull requests. If you haven't signed the CLA, the assistant will provide a link in your PR to do so.

## How to Contribute

1.  **Fork the Repository:** Create your own fork of the ReceiptSorter project.
2.  **Clone your Fork:** `git clone https://github.com/YOUR_USERNAME/ReceiptSorter.git`
3.  **Create a Branch:** Create a feature branch for your changes. `git checkout -b feature/your-feature-name`
4.  **Make Changes:** Implement your changes, following the existing code style and conventions.
5.  **Commit Changes:** Write clear, descriptive commit messages.
6.  **Push to GitHub:** Push your branch to your fork. `git push origin feature/your-feature-name`
7.  **Submit a Pull Request:** Open a PR against the `main` branch of the original repository.

## Contribution Guidelines

- **Code Quality:** Ensure your code is well-formatted and follows the project's established patterns.
- **Testing:** Include tests for any new features or bug fixes whenever possible.
- **Documentation:** Update relevant documentation if your changes introduce new functionality or change existing behavior.

We look forward to your contributions!
